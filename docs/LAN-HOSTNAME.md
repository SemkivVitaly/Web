# Зайти по имени `localchat` вместо IP

Браузер не «знает» имя `localchat` сам по себе: нужно сказать системе, **какой IP** за этим именем стоит. Обычно это делают через файл **hosts**.

## На том же ПК, где крутится Docker (только вы)

1. Откройте блокнот **от имени администратора**.
2. Файл → Открыть:  
   `C:\Windows\System32\drivers\etc\hosts`  
   (внизу списка типов выберите «Все файлы»).
3. В конец файла добавьте строку:

```text
127.0.0.1   localchat
```

4. Сохраните файл.
5. В браузере откройте: **http://localchat:3780**

Имя в адресе — это только удобство; сервер по-прежнему слушает порт **3780** на вашей машине.

## С телефона или другого компьютера в Wi‑Fi

На **каждом** таком устройстве в hosts нужно указать **LAN-IP компьютера с Docker**, не `127.0.0.1`:

```text
192.168.1.50   localchat
```

Подставьте свой IP (в PowerShell на сервере: `ipconfig`, смотрите IPv4 адаптера Wi‑Fi/Ethernet).

На телефоне без root hosts часто недоступен — тогда остаётся IP или свой DNS в роутере (см. ниже).

## CORS (важно)

Сервер разрешает запросы только с перечисленных в **`CORS_ORIGINS`** адресов. Если открываете **http://localchat:3780**, этот origin должен быть в списке.

В `docker-compose.yml` по умолчанию уже добавлен `http://localchat:3780`. Если переопределяете `CORS_ORIGINS` в `.env`, не забудьте включить туда и это имя:

```env
CORS_ORIGINS=http://localchat:3780,http://localhost:3780,http://192.168.1.50:3780
```

После смены переменных: `docker compose up -d --force-recreate localchat`.

## OnlyOffice с других устройств

Если заходите не с localhost, в `.env` для Docker обычно задают доступный с этой сети адрес Document Server, например:

```env
ONLYOFFICE_DOCUMENT_SERVER_URL=http://192.168.1.50:8080
```

(порт **8080** — как в `docker-compose.yml` у сервиса `onlyoffice`).

## Альтернатива: имя на роутере (без hosts на каждом ПК)

Часть роутеров умеет «Статический DHCP» / привязку имени к MAC-адресу ПК. Тогда в сети может появиться имя вроде `my-pc.lan` — зависит от модели. Универсальная схема для учебной сети — **hosts** на нужных машинах.

## Ограничения частоты

Для снижения злоупотреблений включены лимиты (ответ **429**): в том числе вход/регистрация по IP, отправка сообщений и глобальный поиск — по пользователю.
