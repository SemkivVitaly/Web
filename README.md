# LocalChat

Локальный мессенджер для **только LAN**: без обращений к внешнему интернету в работе приложения (после установки зависимостей на машине, где собираете проект).

## Возможности

- **Профиль**: регистрация, вход, имя, био, аватар; **тег** — единственный в системе идентификатор (`UNIQUE` в БД, без учёта регистра): коллеги, приглашения в группы и `@упоминания` только по нему.
- **Группы**: создание, вход по **коду присоединения** и паролю чата, роли (админ / модератор / участник), **приглашения по тегу** пользователя, кик, бан, смена названия и пароля.
- **Упоминания в группе**: текст `@тег` — уведомление адресату (если он в группе).
- **Личные чаты** только между пользователями из списка **коллег**.
- **Вложения**: файлы, изображения, аудио/видео.
- **Закрепление сообщений** (в группе — модераторы и админ; в личном — оба участника).
- **Коллеги**: заявки **по тегу** (`@тег` или без `@`), принятие/отклонение по тегу отправителя, удаление из списка коллег.
- **Список чатов**: закрепление и избранное (сортировка в боковой панели).
- **Совместные документы** (вкладка «Документы» у группы): текстовый редактор на [TipTap](https://tiptap.dev/) + [Yjs](https://yjs.dev/) (как упрощённый Google Docs) и **таблица** 8×20 с синхронизацией ячеек; опциональный **пароль** на документ; состояние хранится на сервере в SQLite (BLOB).
- **Задачи** (вкладка «Задачи»): доски внутри группы (опционально с **паролем**), **иерархия** подзадач, статусы (к выполнению / в работе / на проверке / готово), **прогресс** 0–100 и **эффективный прогресс** родителя (среднее по подзадачам), исполнитель из участников группы, **комментарии**, **фото** во вложениях, обновления в реальном времени через Socket.io.

## Запуск

### Разработка (две консоли)

1. Сервер (слушает `0.0.0.0:3780` — доступен в локальной сети):

```bash
cd server
npm install
npm run dev
```

2. Клиент (прокси к серверу):

```bash
cd client
npm install
npm run dev
```

Откройте в браузере адрес, который покажет Vite (обычно `http://localhost:5173`). Чтобы заходить с **других ПК** в режиме разработки, запустите Vite с хостом `0.0.0.0`: `npm run dev -- --host 0.0.0.0` и откройте `http://<IP>:5173` (прокси `/api` и WebSocket пойдут на тот же ПК). Проще для LAN — собрать клиент и открывать только порт сервера (см. ниже).

### Продакшен на одном ПК (удобно для всей сети)

```bash
cd client && npm install && npm run build
cd ../server && npm install && npm start
```

Откройте `http://<IP-этого-ПК>:3780` с любого устройства в той же сети. Для Docker см. блок **Docker Compose** ниже. Если подключён OnlyOffice **без** Docker и **`PUBLIC_BASE_URL` не задан**, базовый URL для ссылок, которые скачивает Document Server, берётся из заголовка **`Host`** запроса конфига — заход по IP даёт ссылки с IP, а не с `127.0.0.1`.

### Docker Compose

В корне репозитория создайте `.env` и укажите **`LAN_HOST=<IP этого ПК в локальной сети>`** (например `LAN_HOST=192.168.1.10`). Тогда подставятся **CORS** для `http://<LAN_HOST>:3780` и URL OnlyOffice для **браузера** `http://<LAN_HOST>:<ONLYOFFICE_PORT>` (по умолчанию порт **8081** на хосте, т.к. **8080** часто занят; меняется переменной **`ONLYOFFICE_PORT`** в `.env`, в контейнере OnlyOffice по-прежнему слушается **:80** — в `docker-compose` маппинг `ONLYOFFICE_PORT:80`). **`PUBLIC_BASE_URL` по умолчанию остаётся `http://localchat:3780`**: Document Server в Docker ходит к API по **внутренней сети compose** (надёжно). Вариант через `host.docker.internal` на части систем ломает скачивание файлов и сохранение — не используйте его без необходимости. Сервис `onlyoffice` с **`extra_hosts: host.docker.internal:host-gateway`** оставлен на случай, если вы явно зададите `PUBLIC_BASE_URL=http://host.docker.internal:3780` в `.env`. Без `LAN_HOST` в URL редактора остаётся `localhost` — с других устройств в LAN OnlyOffice не откроется.

**Вместо IP — имя `localchat`:** добавьте в файл hosts строку `127.0.0.1 localchat` (на своём ПК) или `192.168.x.x localchat` (на других устройствах в LAN) и откройте `http://localchat:3780`. Подробно: [docs/LAN-HOSTNAME.md](docs/LAN-HOSTNAME.md).

Переменные окружения (опционально):

| Переменная   | Описание |
|-------------|----------|
| `PORT`      | Порт (по умолчанию `3780`) |
| `HOST`      | Интерфейс (по умолчанию `0.0.0.0`) |
| `JWT_SECRET` | Секрет подписи JWT. **В production обязателен** (иначе сервер не стартует). |
| `SQLITE_PATH` | Путь к файлу SQLite БД |
| `NODE_ENV` | Если `production`, требуются `JWT_SECRET` и **`CORS_ORIGINS`** (через запятую, например `https://chat.lan`) |
| `CORS_ORIGINS` | Разрешённые origin для API и Socket.io (в dev можно не задавать) |
| `TRUST_PROXY` | `1` — если сервер за reverse-proxy (nginx): корректные IP в логах и схема/хост для OnlyOffice при авто-`PUBLIC_BASE_URL` |
| `ONLYOFFICE_DOCUMENT_SERVER_URL` | URL OnlyOffice Document Server для браузера; пусто — без внешнего редактора |
| `LAN_HOST` | Только сценарий **Docker Compose**: IP этого ПК в LAN — подставляется в дефолты `CORS_ORIGINS` и `ONLYOFFICE_DOCUMENT_SERVER_URL` в `docker-compose.yml` |
| `ONLYOFFICE_PORT` | Только **Docker Compose**: порт на **хосте** для сервиса `onlyoffice` (по умолчанию **8081**). Проброс: `HOST:80` внутри контейнера. Должен совпадать с портом в `ONLYOFFICE_DOCUMENT_SERVER_URL`, если задаёте URL вручную |
| `PUBLIC_BASE_URL` | Базовый URL вашего API **как его видит Document Server** (скачивание файлов, callback). Если не задан — берётся из запроса к API (`Host` + порт; за прокси с `TRUST_PROXY=1` — `X-Forwarded-Host` / `X-Forwarded-Proto`). В **docker compose** по умолчанию **`http://localchat:3780`** (сеть между контейнерами). Не подменяйте на `host.docker.internal`, если документы перестали открываться |
| `ONLYOFFICE_JWT_SECRET` | Секрет JWT для converter OnlyOffice (если включён JWT на стороне DS) |
| `COLLAB_POOL_MAX` | Лимит документов Yjs в RAM на сервере (по умолчанию `40`) |
| `COLLAB_MAX_UPDATE_B64` | Макс. длина base64 одного апдейта collab (по умолчанию ~1.2M символов) |

**Безопасность:** в релизе задайте `NODE_ENV=production`, `JWT_SECRET` и `CORS_ORIGINS`. Раньше любой залогиненный пользователь мог подписаться на Socket.io-комнату чужого чата по ID — сейчас сервер проверяет членство в группе / участие в личном диалоге.

## Архитектура

- **Backend**: Node.js, Express, Socket.io, SQLite (better-sqlite3), загрузки на диск в `server/uploads/`.
- **Frontend**: React + Vite, Socket.io-client.

Данные и файлы остаются на машине, где запущен сервер.
docker compose up --build -d
